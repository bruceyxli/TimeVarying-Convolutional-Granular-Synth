Time‑Varying Convolutional Granular Synth (Python)
Bruce Li — Music 159

Purpose (What and Why)
• Fuse granular synthesis with time‑varying convolution using very short impulse responses (IRs), so timbre can evolve at the grain rate without smearing attacks.  
• Offer practical variants for different musical goals and a simple, reliable workflow (Streamlit GUI + CLI).  
• Typical musical uses: evolving pads and drones; percussive “micro‑rooms”; vocal/instrument timbre morphing; cinematic soundscapes; transitions/risers; creative short‑IR “chorus‑reverbs”; live performance with mapped controls.

High‑Level Design (How it works)
• Pipeline: Source → Grain Renderer (window, pitch, pan, jitter) → Per‑grain Convolution → Wet/Dry Mix → Normalization → WAV export.  
• Micro‑IRs are short (8–32 ms). Each grain can pick a different IR using a strategy (fixed / cycle / random / weighted / centroid). This makes the effective kernel time‑varying at the grain rate.  
• All processing is local/offline. The GUI provides a single‑page control surface; the CLI exposes the same engine for batch rendering.

Variants (Sound character and use cases)
• Standard: Per‑grain short‑IR convolution. Produces time‑varying color with crisp onsets. Good general default for pads, plucks with space, and animated textures.  
• Variant A (Convolve → Granulate): Convolve the entire source once with a longer IR (e.g., 120 ms) and then granulate without per‑grain convolution. More coherent, less flicker—good for steady‑state material.  
• Variant B (Grains as IR): Each grain acts as a short IR while the local source segment is the exciter. Distinct “micro‑body” color that moves at the grain rate—great for percussive ear‑candy and tone‑design.

Micro‑IR Bank (Generation and selection)
• Auto‑generated by default: white‑noise impulse → random bandpass → exponential decay → peak alignment (t=0) → peak normalization. Length 8–32 ms; bank size 32–128.  
• Selection strategies:  
  – fixed: always IR[0]  
  – cycle: round‑robin  
  – random / weighted: stochastic (weighted supports arbitrary distributions)  
  – centroid: choose IR whose spectral centroid is closest to the current grain’s centroid

Key Techniques (Engineering details)
• Windowed grains (Hann) to avoid clicks; quasi‑synchronous triggering with jitter for natural variation.  
• Pitch via resampling (resample_poly) with fixed output length to keep timing consistent.  
• Per‑grain spectral centroid for content‑aware IR selection.  
• Fast convolution (FFT). Short IRs keep computation low while enabling rapid color changes.  
• Streamlit UI designed to be session‑state‑safe; single‑page layout with sensible defaults.  
• Reproducibility: all randomness is seeded (default seed = 2025).

Installation
1) Install Python dependencies:
   pip install -r requirements.txt

How to Use — GUI (Recommended)
1) Launch the app:
   streamlit run src/app/ui.py
2) In the browser, optionally upload audio (otherwise a demo source is generated).  
3) Set Variant, Sample Rate, Duration, and the Wet/Dry balance in “Global.”  
4) Shape “Grain” (Density, Grain Length, Trigger Jitter, Pitch Range, Pan Spread).  
5) Configure “IR” (IR Length, Bank Size, Selection strategy, Random Seed).  
6) Click “Render,” preview the result, then “Download WAV.”  
Notes: Duration auto‑aligns to uploaded audio length (clamped 2–40 s). Applying a preset does not change Duration.

How to Use — CLI
• CLI support has been removed in this submission. Please use the Streamlit GUI workflow above.

Tips for Musical Results
• Denser space: longer/more IRs and higher density.  
• Clearer attacks: shorter IR + lower Wet + moderate jitter.  
• Wider stereo: increase Pan Spread; keep Wet controlled to avoid wash.  
• Adaptive timbre: use “centroid” selection so IR color follows the grain’s spectrum.  
• Variant A for coherent ambience; Variant B for pronounced granular coloration.

Troubleshooting
• If the browser doesn’t open, copy the local URL printed by Streamlit and paste it into your browser.  
• Large uploads: keep files short (≤ 40 s target) for quick iteration.  
• If Duration slider resists changes after upload, it is auto‑aligned by design; upload a different file or adjust Sample Rate to re‑compute the alignment, then move the slider as needed (within 2–40 s).

Project Structure
• src/app/engine.py: granular engine, variants, rendering pipeline.  
• src/app/features.py: spectral centroid.  
• src/app/ir_bank.py: micro‑IR generation / loading / selection.  
• src/app/presets.py: preset definitions.  
• src/app/ui.py: Streamlit GUI.
